# 🏋️ Aesthetix — Тренувальний компаньйон (David Laid Style)

Це сучасний вебдодаток для відстеження прогресу тренувань за 12-тижневою періодизованою бодібілдерською програмою. Дизайн виконано в стилі **Apple iOS** (шрифт San Francisco, Blur-ефекти, закруглення iOS, адаптація під iPhone 12).

---

## 🚀 Як викласти на GitHub та розгорнути на Vercel

### Крок 1: Завантаження коду на GitHub
1. Зайди на [GitHub](https://github.com/) та створи новий **публічний або приватний репозиторій** (наприклад, з назвою `aesthetix-gym`).
2. Відкрий термінал на комп'ютері та перейди в папку з проектом на Робочому столі:
   ```bash
   cd ~/Стільниця/aesthetix-github-deploy
   ```
3. Ініціалізуй git, додай файли та зроби перший комміт:
   ```bash
   git init
   git add .
   git commit -m "Initial commit of Aesthetix Gym app"
   ```
4. Зв'яжи локальний проект із твоїм репозиторієм на GitHub (заміни `ТВІЙ_ЮЗЕРНЕЙМ` на свій логін GitHub):
   ```bash
   git branch -M main
   git remote add origin https://github.com/ТВІЙ_ЮЗЕРНЕЙМ/aesthetix-gym.git
   git push -u origin main
   ```

---

### Крок 2: Деплой на Vercel
1. Перейди на [Vercel](https://vercel.com/) та увійди під своїм акаунтом GitHub.
2. Натисни кнопку **«Add New»** → **«Project»**.
3. Знайди свій репозиторій `aesthetix-gym` у списку та натисни **«Import»**.
4. У налаштуваннях проекту **нічого міняти не потрібно** (файл `vercel.json` вже налаштований для серверної обробки Express).
5. Натисни кнопку **«Deploy»**. За 1 хвилину додаток буде запущено, і Vercel видасть тобі публічне посилання (наприклад, `https://aesthetix-gym.vercel.app`), яке ти зможеш відкрити на своєму iPhone!

---

## 📦 Опис файлової структури проекту:
- `vercel.json` — Файл конфігурації для Vercel. Він каже платформі використовувати `@vercel/node` для обробки серверної частини `server.js` і маршрутизувати всі запити `/api/...` на неї, а решту запитів — віддавати як статичні файли з папки `public`.
- `server.js` — Серверна частина на Express. Вона керує входом/реєстрацією користувачів, хешуванням паролів за допомогою PBKDF2 та збереженням прогресу користувачів.
- `package.json` — Список залежностей та налаштування запуску проекту.
- `public/index.html` — Весь клієнтський додаток (HTML, CSS, JS). Працює як Single Page Application. Має вбудовану офлайн-підтримку через `localStorage` і звуковий рушій для таймерів.

## 💡 Важлива інформація про роботу на Vercel
Серверні функції на Vercel є **Serverless** (безсерверними) і мають файлову систему, яка доступна лише для читання. Наш `server.js` оптимізовано під це:
- На Vercel файли збереження записуються в директорію `/tmp` (це єдина папка, куди Vercel дозволяє записувати дані).
- Дані у папці `/tmp` є тимчасовими і видаляються, коли серверна функція «засинає». 
- **Але твій додаток не втратить прогрес!** Клієнтська частина автоматично зберігає весь твій стан, історію та 1RM у `localStorage` браузера на твоєму iPhone. Навіть якщо сервер Vercel буде перезапущено, твій телефон відновить локальну копію та знову синхронізує її.
